# Playbook Index

מבנה קצרצר של 15 העקרונות. מיועד **רק** ל-Mapper LLM. אסור להעביר אותו ל-Extractor (שאסור לדעת על הפלייבוק) ואסור להחליף בו את הפלייבוק המלא ל-Synthesizer (שצריך גישה ל-Evidence Signatures, Decision Rules, ו-Disqualifiers).

הפורמט: `Pxx Name — short phenomenon (≤12 words)`. בלי evidence signatures, בלי decision rules, בלי disqualifiers.

---

```
P01 Action-Defeats-Helplessness — short measurable wins restore buyer agency
P02 Explanation-Efficiency — buyer can repeat the offer in one sentence
P03 Cognitive-Frame-Shift — CEO reframes buyer's mental model of the problem
P04 Tradeoff-Declaration — explicit non-customer / what we don't do
P05 Mechanism-Articulation — named methodology / "we do X through Y"
P06 Time-to-First-Value — measurable buyer-reported outcome within ≤30 days
P07 Decision-Latency-Reduction — committee approves in fewer meetings
P08 Cycle-Compression-Through-Templating — repeated framework shortens engagements
P09 Domain-Empathy-Translation — vendor speaks buyer's industry language natively
P10 Onboarding-Velocity — buyer team productive in days, not weeks
P11 Risk-Reversal — pilot / exit clause / phased rollout shifts risk to vendor
P12 Loss-Avoided-Quantification — $ figure on cost-of-doing-nothing
P13 Competitive-Displacement-Evidence — named competitor switch documented
P14 Pricing-Power-from-Niche — narrow ICP enables premium pricing
P15 Legitimacy-Driver — known brand / certification / advisory used as $ proxy
```

## CTM axis labels (for Mapper context only — NOT a mapping signal)

```
M (Mental):  P01 P02 P03 P04 P05
T (Time):    P06 P07 P08 P09 P10
C (Cash):    P11 P12 P13 P14 P15
```

ה-Mapper יכול להשתמש בלייבל הזה כדי להבין את ה-grouping, אבל אסור לו למפות "כל מה שקשור לזמן → T-axis principles". המיפוי חייב להיות ספציפי ל-principle, לא ל-axis.
